# Plan: Release v1.1.0 — 7 mejoras de producto

## Contexto

El proyecto está actualmente en producción en Vercel (v1.0.0, tagged en GitHub). Se quiere lanzar una nueva versión con 7 mejoras agrupadas en ramas de feature según el flujo definido en `rules/github-flow.md` y `rules/deploy-guide.md`. Todas las features siguen los patrones existentes del proyecto: Server Actions + `revalidatePath`, componentes `*PageClient`, Chakra UI dark theme, Zod validation.

---

## Diagnóstico previo (hallazgos de exploración)

- **Bug de estado**: `TransactionsPageClient` tiene `const [transactions] = useState(initialTransactions)` (sin setter ni `useEffect`), por lo que `router.refresh()` no actualiza la lista en pantalla.
- **Token en cookies**: Los tokens visibles son exclusivamente tokens estándar de NextAuth (`csrf-token`, `callback-url`, `session-token` en formato JWE/A256GCM). El `INSFORGE_API_KEY` **no está expuesto** — solo se usa en `lib/insforge-admin.ts` (server-side). Los tokens vistos son seguros y esperados.
- **No existen tablas `accounts` ni `account_movements`** — son entidades nuevas que requieren creación en InsForge.
- **FloatingChat**: El panel usa `bottom={{ base: '80px' }}` y el botón `bottom={{ base: '72px' }}` — necesita más separación en mobile para no solaparse.

## Feature 1: Auto-reload de transacciones (`feature/1-transaction-sync`)

**Problema:** `router.refresh()` re-renderiza el Server Component pero el Client Component no actualiza su estado porque usa `useState` sin sync con el prop.

**Archivos a modificar:**
- `app/(dashboard)/transactions/TransactionsPageClient.tsx`
- `components/transactions/TransactionForm.tsx`
- `components/transactions/TransactionEditForm.tsx`
- `components/dashboard/RecentTransactions.tsx`
- `components/chat/ChatInterface.tsx`

**Cambios:**

1. **`TransactionsPageClient.tsx`**: Cambiar `const [transactions] = useState(...)` a `const [transactions, setTransactions] = useState(...)` y agregar:
   ```tsx
   useEffect(() => {
     setTransactions(initialTransactions)
   }, [initialTransactions])
   ```

2. **`TransactionForm.tsx` y `TransactionEditForm.tsx`**: Verificar que llaman `router.refresh()` después del Server Action exitoso.

3. **`ChatInterface.tsx`**: Después de confirmar la transacción creada por IA, llamar `router.refresh()`.

4. **`RecentTransactions.tsx`**: Si recibe props estáticas, aplicar mismo patrón de `useEffect` sync.

## Feature 2: Settings con Tabs + Cuentas + Movimientos (`feature/2-accounts-settings`)

### 2a. Nuevas tablas en InsForge (paso manual previo a código)

**Tabla `accounts`:**
```sql
id          uuid (PK, default gen_random_uuid())
user_id     uuid (FK → users.id)
name        text
type        text (bank | digital | crypto | cash)
currency    text (COP | USD | VES)
balance     numeric (default 0)
color       text (hex, nullable)
icon        text (emoji, nullable)
is_active   boolean (default true)
created_at  timestamptz (default now())
updated_at  timestamptz (default now())
```

**Tabla `account_movements`:**
```sql
id               uuid (PK)
user_id          uuid (FK → users.id)
from_account_id  uuid (FK → accounts.id)
to_account_id    uuid (FK → accounts.id)
amount           numeric
currency         text
description      text (nullable)
date             date
created_at       timestamptz (default now())
```

**Modificar `transactions`:** Agregar `account_id uuid (FK → accounts.id, nullable)`

### 2b–2d. Tipos + Validaciones + Server Actions

- `lib/actions/account_movements.actions.ts` — (getAccountMovements, createAccountMovement, deleteAccountMovement)

### 2e. Componentes UI

- `components/settings/AccountForm.tsx` — reusar FormDialog + FormInput + IconPicker + ColorPicker

- `components/settings/AccountsTab.tsx` — lista de cuentas + CRUD + tabla de movimientos

### 2f. Settings con Tabs

- `app/(dashboard)/settings/page.tsx` — agregar fetch de getAccounts + getAccountMovements

### 2g. Dashboard — cards de cuentas

## Deploy Flow

### Pre-deploy
1. Crear tablas en InsForge dashboard
2. `pnpm build` exitoso
3. Actualizar env vars si es necesario

### Por cada feature
```bash
git checkout develop && git pull origin develop
git checkout -b feature/N-description
# implementar
git commit -m "feat: descripcion"
git push origin feature/N-description
# PR → develop
```

### Release
```bash
git checkout main
git merge develop
git push origin main
git tag v1.1.0 && git push origin v1.1.0
```
